# DSAForge — Railway Deployment Guide

Complete step-by-step guide to deploy the full product (React + FastAPI + PostgreSQL)
on Railway. Takes about 20 minutes.

---

## WHAT RAILWAY GIVES YOU

- PostgreSQL database (managed, auto-backups)
- Backend service (Python FastAPI, auto-deploy from GitHub)
- Frontend service (React, static build)
- Free HTTPS on every service
- Automatic redeploy on every `git push`
- $5 free credits/month on Hobby plan (enough for light usage)

---

## BEFORE YOU START — Things you need

- [ ] A GitHub account
- [ ] The dsaforge project folder (unzipped)
- [ ] 20 minutes

---

## STEP 0 — Push your code to GitHub

Open terminal inside your dsaforge folder:

```bash
cd dsaforge

# Initialize git
git init
git add .
git commit -m "DSAForge initial commit"
```

Now go to **github.com** → click the **+** icon → **New repository**
- Repository name: `dsaforge`
- Set to **Public** (or Private, both work)
- Do NOT add README or .gitignore (your code already has these)
- Click **Create repository**

Back in your terminal:
```bash
git remote add origin https://github.com/YOUR_USERNAME/dsaforge.git
git branch -M main
git push -u origin main
```

✅ Your code is now on GitHub.

---

## STEP 1 — Create Railway Account

1. Go to **https://railway.app**
2. Click **Login** → **Login with GitHub**
3. Authorize Railway
4. You're in. Railway gives you **$5 free credits** — enough to test the full app.

For paid usage: Hobby plan is **$5/month** which gives you more resources. Most small apps run fine on free credits for the first month.

---

## STEP 2 — Create a New Project

1. On the Railway dashboard, click **New Project**
2. You'll see options — click **Empty Project**
3. Your project is created. It's empty for now — you'll add 3 services to it.

---

## STEP 3 — Add PostgreSQL Database

Inside your project:

1. Click **+ Add Service** (or the **+** button)
2. Click **Database**
3. Click **Add PostgreSQL**
4. Railway creates the database in ~30 seconds

Once created, click on the **Postgres** service → go to the **Variables** tab.

You'll see these variables auto-generated by Railway:
```
DATABASE_URL          postgresql://postgres:xxx@xxx.railway.internal:5432/railway
DATABASE_PRIVATE_URL  postgresql://postgres:xxx@xxx.railway.internal:5432/railway
DATABASE_PUBLIC_URL   postgresql://postgres:xxx@xxx.up.railway.app:PORT/railway
PGHOST                xxx.railway.internal
PGPORT                5432
PGDATABASE            railway
PGUSER                postgres
PGPASSWORD            your-generated-password
```

⚠️ **Important:** Copy the `PGPASSWORD` value — you'll need it in Step 4.

---

## STEP 4 — Deploy the Backend (FastAPI)

Still inside your Railway project:

1. Click **+ Add Service**
2. Click **GitHub Repo**
3. Select your `dsaforge` repository
4. Railway asks which folder to deploy — type: `backend`
   (or it may auto-detect it — select the one with the Dockerfile)
5. Click **Add Service** / **Deploy**

Railway will start building. While it builds, set the environment variables:

Click on your new **backend** service → **Variables** tab → click **New Variable**

Add these one by one:

| Variable Name | Value |
|---|---|
| `DATABASE_URL` | `${{Postgres.DATABASE_URL}}` |
| `SECRET_KEY` | `dsaforge-super-secret-key-change-this-2024xyz` |
| `PORT` | `8000` |

**How to add `DATABASE_URL` using Railway's reference system:**
- Click **New Variable**
- Name: `DATABASE_URL`
- For the value, type `${{` and Railway will show an autocomplete dropdown
- Select `Postgres` → select `DATABASE_URL`
- This links it automatically — Railway fills in the real connection string

Click **Deploy** to redeploy with the new variables.

Wait ~2-3 minutes for the build to finish.

**Check if backend is working:**
Click on your backend service → **Settings** → **Networking** → click **Generate Domain**

You'll get a URL like:
```
https://dsaforge-backend-production.up.railway.app
```

Open that URL in your browser. You should see:
```json
{"message": "DSAForge API running ⚡"}
```

Also test: `https://YOUR-BACKEND-URL.up.railway.app/health`
Should return: `{"status": "ok"}`

✅ Backend is live. Copy this URL — you'll need it for the frontend.

---

## STEP 5 — Deploy the Frontend (React)

1. Click **+ Add Service** again
2. Click **GitHub Repo** → same `dsaforge` repo
3. This time set root directory to: `frontend`
4. Click **Add Service**

While it builds, add the environment variable:

Click on **frontend** service → **Variables** → **New Variable**

| Variable Name | Value |
|---|---|
| `VITE_API_URL` | `https://YOUR-BACKEND-URL.up.railway.app` |
| `PORT` | `3000` |

Replace `YOUR-BACKEND-URL` with the actual backend URL from Step 4.

Example:
```
VITE_API_URL = https://dsaforge-backend-production.up.railway.app
```

Click **Deploy** to rebuild with this variable.

Wait ~3-4 minutes (React build takes a bit longer).

Once done, go to frontend service → **Settings** → **Networking** → **Generate Domain**

You'll get:
```
https://dsaforge-frontend-production.up.railway.app
```

Open this in your browser. **Your app is live!**

---

## STEP 6 — Test Everything Works

Open your frontend Railway URL in browser.

Run through this checklist:

```
[ ] Register page loads
[ ] Can create a new account
[ ] Taken to onboarding (AI connection step)
[ ] Can skip AI connection
[ ] Evaluation step appears
[ ] Can pick a level and submit
[ ] Dashboard loads with 0 companies
[ ] Can click Practice → questions load
[ ] Can mark a question as Solved → stays solved after page refresh
[ ] Dashboard shows solved count updating
```

If the frontend loads but API calls fail, check:
- Frontend's `VITE_API_URL` variable matches the exact backend URL
- Backend service is running (green dot in Railway)
- Backend logs show no startup errors (click service → Logs tab)

## STEP 7 — Connect a Custom Domain (Optional)

If you have a domain (e.g. `dsaforge.com` or `yourdomain.in`):

**On Railway:**
1. Click your frontend service → **Settings** → **Networking**
2. Under **Custom Domain** → type your domain → click **Add**
3. Railway shows you DNS records to add

**On your domain registrar (GoDaddy / Namecheap / etc.):**
1. Go to DNS settings
2. Add the CNAME record Railway shows you:
   ```
   Type: CNAME
   Name: @  (or www)
   Value: dsaforge-frontend-production.up.railway.app
   TTL: 3600
   ```
3. Wait 5-30 minutes for DNS to propagate

Railway automatically gives you free HTTPS (SSL certificate) once the domain is connected.

## STEP 8 — Auto-Deploy on Code Changes

Railway already set this up automatically. Every time you push to GitHub:

```bash
# Make a change to your code, then:
git add .
git commit -m "your change"
git push origin main
```

Railway detects the push and redeploys the affected service automatically. You can watch it build live in the Railway dashboard.

---

## STEP 9 — View Logs (for debugging)

If something breaks:

1. Click the service that's failing
2. Click **Logs** tab
3. You'll see real-time logs

Common issues and fixes:

**Backend won't start:**
```
Error: could not connect to server: Connection refused
```
→ Check that `DATABASE_URL` variable is set correctly with `${{Postgres.DATABASE_URL}}`

**Frontend shows blank page:**
→ Check browser console (F12) for errors
→ Most likely `VITE_API_URL` is wrong — should be `https://...` not `http://`

**API calls return 404:**
→ Check that `VITE_API_URL` does NOT have a trailing slash
→ Correct: `https://backend.up.railway.app`
→ Wrong: `https://backend.up.railway.app/`

**CORS errors in browser:**
→ This is handled. If you still see them, check backend is running at the exact URL in `VITE_API_URL`

## STEP 10 — Upgrade When You Have Users

Railway free credits ($5/month) covers roughly:
- ~500 MB RAM usage
- ~100 requests/hour consistently

When your app grows:

1. Go to Railway → Account Settings → **Billing**
2. Upgrade to **Hobby Plan** ($5/month flat + usage)
3. This gives you persistent deployments (free tier services sleep after 30 days of inactivity)

The Hobby plan easily handles **1,000–5,000 users** for under $20/month.
